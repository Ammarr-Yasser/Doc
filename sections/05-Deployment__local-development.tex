\hypertarget{purpose}{%
\subsubsection{Purpose}\label{sec:05-Deployment__local-development:purpose}}

This document describes how to run DefendX in a \textbf{local development environment} for testing, debugging, and development.

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\hypertarget{development-characteristics}{%
\subsubsection{Development Characteristics}\label{sec:05-Deployment__local-development:development-characteristics}}

Local development environments typically enable:

\begin{itemize}
\tightlist
\item
  Debug logging
\item
  Local databases
\item
  Relaxed CORS rules
\item
  Faster iteration cycles
\end{itemize}

These settings must not be used in production.

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\hypertarget{backend-setup-local}{%
\subsubsection{Backend Setup (Local)}\label{sec:05-Deployment__local-development:backend-setup-local}}

Typical local workflow:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Create and activate a virtual environment
\item
  Install backend dependencies
\item
  Set required environment variables
\item
  Initialize the database
\item
  Start the backend server
\end{enumerate}

The backend exposes:

\begin{itemize}
\tightlist
\item
  API endpoints under \texttt{/api}
\item
  Swagger documentation under \texttt{/docs}
\end{itemize}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\hypertarget{frontend-setup-local}{%
\subsubsection{Frontend Setup (Local)}\label{sec:05-Deployment__local-development:frontend-setup-local}}

Typical frontend workflow:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Install frontend dependencies
\item
  Configure API base URL
\item
  Start the development server
\item
  Interact with the backend API
\end{enumerate}

Frontend and backend usually run on separate ports during development.

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\hypertarget{testing-tools-locally}{%
\subsubsection{Testing Tools Locally}\label{sec:05-Deployment__local-development:testing-tools-locally}}

\begin{itemize}
\tightlist
\item
  Use benign test URLs and files
\item
  Avoid submitting sensitive personal data
\end{itemize}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\hypertarget{common-development-pitfalls}{%
\subsubsection{Common Development Pitfalls}\label{sec:05-Deployment__local-development:common-development-pitfalls}}

\begin{itemize}
\tightlist
\item
  Missing environment variables
\item
  CORS misconfiguration
\item
  External API rate limits
\end{itemize}

\begin{center}\rule{0.5\linewidth}{0.5pt}\end{center}

\hypertarget{development-safety}{%
\subsubsection{Development Safety}\label{sec:05-Deployment__local-development:development-safety}}

Local development should:

\begin{itemize}
\tightlist
\item
  Avoid real user data
\item
  Use isolated test accounts
\item
  Never expose development services publicly
\end{itemize}
